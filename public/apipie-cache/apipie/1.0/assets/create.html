POST /api/v1/assets
creates a new asset

Examples

Sample Request:

{
    "name": "A sample asset",
    "listing_order": 0,
    "submission_id": 7395,
    "external_source": "youtube",
    "external_location": "PiqepG5GDZI"
}

Sample Response:

{
    "id": 4544,
    "family": "video",
    "type": "Asset::Youtube",
    "name": "A sample asset",
    "created_at": "May 05 2015",
    "attachment_state": "processed",
    "thumbnail": "http://img.youtube.com/vi/PiqepG5GDZI/mqdefault.jpg",
    "locations": {
        "url": "http://www.youtube.com/watch?v=PiqepG5GDZI"
    },
    "file_info": "Youtube Video",
    "listing_order": 4194304,
    "submission_id": 7395,
    "creator_type": "User",
    "project_id": 466,
    "creator_id": 4483,
    "annotations_count": 0,
    "attachment": null,
    "annotations": []
}

Errors

  • 403: The user is not authorized to attach an asset to this submission.
  • 400: A required asset parameter is missing.

Request Parameters

Param name Description
id
optional

The asset ID


Value:

Must be Fixnum
submission_id
required

The ID of the submission with which this asset is associated


Value:

Must be Fixnum
listing_order_position
optional

The position of this asset among other assets associated with the same submission


Value:

Must be Fixnum
name
optional

The name of the asset


Value:

Must be String
external_source
optional

If the asset references an external asset, the external_source should specify what service hosts the asset


Value:

Must be one of: youtube, vimeo.
external_location
optional

The identifier of the asset at an exteranl source. For example, if the asset is a YouTube video, this would be the YouTube ID


Value:

Must be String
attachment_attributes
optional

Attachment attributes


Value:

Must be a Hash
attachment_attributes[id]
optional

The ID of an attachment object that should be attached to this asset


Value:

Must be Fixnum
