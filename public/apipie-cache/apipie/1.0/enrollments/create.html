POST /api/v1/enrollments
enrolls a single user in a single course

Examples

Sample Request:

{
    "course": 234,
    "user": 4472,
    "role": "evaluator"
}

Sample Response:

{
    "id": "u4472c235",
    "user": 4472,
    "email": "assistant1@test.com",
    "last_sign_in_at": "never",
    "role": "evaluator",
    "course": 235,
    "user_name": "Gusikowski, Toni",
    "course_name": "[ACCMR] ENG2100: Composition I, Spring 2015",
    "section": "ACCMR"
}

Errors

  • 403: Authenticated user is not authorized to mange enrollment for this course.
  • 422: Unable to add the user to the course, perhaps because the user is already enrolled

Request Parameters

Param name Description
course
required

The course's ID


Value:

Must be Fixnum
user
required

The user's ID


Value:

Must be Fixnum
role
required

The course role that will be assigned to the user


Value:

Must be one of: evaluator, assistant, creator.
