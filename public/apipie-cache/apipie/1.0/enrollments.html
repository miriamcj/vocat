Enrollments

Enrollments are a psuedo-resource that represents a user's membership in a given course. In general, course enrollment can be managed by course evaluators, course assistants, and Vocat administrators. Bulk enrollment is possible using the bulk enrollment endpoint. During bulk enrollment, users who are not currently known to Vocat may be invited via email.

>>>

GET /api/v1/courses/:course_id/enrollments?role=:role
returns users enrolled in a given course

GET /api/v1/users/:user_id/enrollments
returns courses in which a given user is enrolled

Only admins and evaluator users may see what courses a user is enrolled in. Any user who is enrolled in a course may view which other users are enrolled in that course.

DELETE /api/v1/enrollments/:enrollment_key
removes a user from a course

Only course evaluators/assistants and Vocat administrators may remove users from courses.

Errors

  • 422: Unable to remove the user from the course
  • 403: Authenticated user is not authorized to mange enrollment for this course.

Request Parameters

Param name Description
enrollment_key
required

The enrollment key is a string that takes the format of uXcY where X is the ID of the user to be de-enrolled, and Y is the ID of the course from which to de-enroll the user. If, for example, you wanted to remove User #10 from Course #234, the enrollment key would be u10c234


Value:

Must be String
>>>

POST /api/v1/enrollments
enrolls a single user in a single course

Examples

Sample Request:

{
    "course": 234,
    "user": 4472,
    "role": "evaluator"
}

Sample Response:

{
    "id": "u4472c235",
    "user": 4472,
    "email": "assistant1@test.com",
    "last_sign_in_at": "never",
    "role": "evaluator",
    "course": 235,
    "user_name": "Gusikowski, Toni",
    "course_name": "[ACCMR] ENG2100: Composition I, Spring 2015",
    "section": "ACCMR"
}

Errors

  • 403: Authenticated user is not authorized to mange enrollment for this course.
  • 422: Unable to add the user to the course, perhaps because the user is already enrolled

Request Parameters

Param name Description
course
required

The course's ID


Value:

Must be Fixnum
user
required

The user's ID


Value:

Must be Fixnum
role
required

The course role that will be assigned to the user


Value:

Must be one of: evaluator, assistant, creator.

POST /api/v1/courses/:course_id/enrollments/bulk
bulk enrolls multiple users in a course

Examples

Sample Request:

{
    "user": 4472,
    "role": "creator",
    "invite": true,
    "contacts": "Jimbo Baggins, jim@castironcoding.com\nBoomBoom Robinson, boomboom@vocat.io"
}

Sample Response:

[
   {
      "email":"jim@castironcoding.com",
      "first_name":"Jimbo",
      "last_name":"Baggins",
      "action":"invite",
      "success":true,
      "reason":null,
      "message":"Jimbo Baggins has been enrolled in ENG2100: Composition I, Section ACCMR",
      "string":"Jimbo Baggins, jim@castironcoding.com"
   },
   {
      "email":"boomboom@vocat.io",
      "first_name":"BoomBoom",
      "last_name":"Robinson",
      "action":"invite",
      "success":true,
      "reason":null,
      "message":"BoomBoom Robinson has been enrolled in ENG2100: Composition I, Section ACCMR",
      "string":"BoomBoom Robinson, boomboom@vocat.io"
   }
]

Errors

  • 403: Authenticated user is not authorized to mange enrollment for this course.

Request Parameters

Param name Description
course_id
required

The course's ID


Value:

Must be Fixnum
role
required

The course role that will be assigned to the user


Value:

Must be one of: evaluator, assistant, creator.
invite
optional

Whether or not to invite users who do not exist in Vocat. If false, users who do not exist will not be invited.


Value:

Must be one of: true, false.
contacts
required

Each line of the string represents a single user to enroll in the course. Each line can be formatted as “FirstName LastName, Email” or “FirstName LastName <email>”


Value:

Must be String
