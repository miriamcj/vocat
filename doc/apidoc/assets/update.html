PATCH /api/v1/assets/:id
updates an asset

Examples

Sample Request:

{
    "name": "A sample asset",
    "listing_order": 0,
    "submission_id": 7395,
    "external_source": "youtube",
    "external_location": "PiqepG5GDZI"
}

Errors

  • 403: The user is not authorized to update this asset.
  • 400: A required asset parameter is missing.

Request Parameters

Param name Description
id
optional

The asset ID


Value:

Must be Fixnum
submission_id
optional

The ID of the submission with which this asset is associated


Value:

Must be Fixnum
listing_order_position
optional

The position of this asset among other assets associated with the same submission


Value:

Must be Fixnum
name
optional

The name of the asset


Value:

Must be String
external_source
optional

If the asset references an external asset, the external_source should specify what service hosts the asset


Value:

Must be one of: youtube, vimeo.
external_location
optional

The identifier of the asset at an exteranl source. For example, if the asset is a YouTube video, this would be the YouTube ID


Value:

Must be String
attachment_attributes
optional

Attachment attributes


Value:

Must be a Hash
attachment_attributes[id]
optional

The ID of an attachment object that should be attached to this asset


Value:

Must be Fixnum
