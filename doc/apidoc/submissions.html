Submissions

Submissions tie together the media, discussion posts, and evaluations that are created in response to a student's work on a project. Submissions are created automatically, on the fly by Vocat. Vocat will always return a one submission for every group and student that can submit for a given project. The existence of a submission, then, should not be taken to mean that a student has taken action on a project. For performance reasons, when the API renders collections of submissions, it uses a brief serializer that does not include all submission details. To access the full details of a submission, API clients should request the single submission.

>>>

GET /api/v1/submission/for_course?course=:course
returns summaries of all submissions for the given course

>>>

POST /api/v1/course/:course_id/submissions DEPRECATED
creates a submission

>>>

PATCH /api/v1/submissions/:id DEPRECATED
updates a submission

>>>

DELETE /api/v1/submissions/:id
deletes a submission and all attached media, annotations, discussion posts, and evaluations.
